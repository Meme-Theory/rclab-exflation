## I. Overview

This document describes the design, methodology, and operational techniques of a multi-agent AI research system built to conduct adversarial theoretical physics research at machine speed. The system was constructed in approximately two weeks by a solo operator — a Navy contractor with a mathematics degree — and has produced eighteen formal research sessions, sixty-seven verification checks at machine epsilon, multiple novel theoretical observations, and a pre-registered computational testing program with calibrated Bayesian update methodology.

The core innovation is not prompt engineering. It is what might be called *epistemological architecture*: the deliberate construction of contexts in which latent mathematical connections across the LLM training corpus are forced to surface, tested adversarially, and either validated or closed by pre-registered criteria. The system treats large language models not as answer machines but as **resonance detectors across the mathematical corpus** — pattern-matching engines whose pattern-matching capacity is exploited rather than dismissed, because mathematics itself is pattern matching.

The framework is designed around a single meta-principle: **build a system to prove yourself wrong**. Every structural choice — adversarial agent mandates, pre-registered Constraint Conditions, gated computational pipelines, honest reporting of negative results — serves this principle. The system's continued survival after eighteen sessions of active attempts at self-destruction is the primary (though not conclusive) evidence for the framework it investigates.

## II. The Operator Model

The Principal Investigator (PI) does not do physics in this system. The PI operates the system.

This is a deliberate architectural choice. The PI's role is to:

1. **Design context windows** — curate which papers, agents, and prior results enter each session
2. **Orchestrate agent interactions** — decide who talks to whom, in what order, with what mandates
3. **Intervene at inflection points** — the PI's two-word mid-session redirections (e.g., "False Vacuum?") have repeatedly reoriented entire research threads more productively than hours of agent discussion
4. **Accept or reject Constraint Conditions** — the PI does not overrule closes, but decides whether the correct quantities were computed
5. **Maintain institutional memory** — compress session outputs into reusable handouts for subsequent sessions

The PI never says "here is what I think the answer is." The PI says: "what do the connective tissues of 2,000 years of mathematics say when tugged this way?" The physics emerges from the agents' interaction with the curated mathematical corpus, not from the operator's prior beliefs.

This creates an unusual epistemological structure: the PI has strong intuitions about the framework (phonon-exflation cosmology), but has constructed a system specifically designed to prevent those intuitions from contaminating the results. The system can only confirm the framework by failing to destroy it through rigorous adversarial testing.

### 3.2 Personality Calibration as Context Engineering

Each agent's system prompt is not merely a role description. It is a *context engineering instrument* that activates specific regions of the LLM's training data.

When Tesla-Resonance is told "you think in harmonics" and given a folder of acoustic physics papers alongside Kaluza-Klein geometry papers, the model is forced to search for connections between those domains. The connections it finds are not hallucinated — they are latent structural relationships in mathematics that exist in the training data but are never activated by conventional queries because no one asks the question that bridges the domains.

This is the key insight: **the training data already contains the connective tissue of mathematics**. Connes and Volovik and Baptista and random matrix theory and structural acoustics all exist in the corpus. They sit in separate compartments because no one asks the question that connects them. The agent architecture creates contexts where those connections are forced to surface.

The personality calibration serves a second function: it creates *productive disagreement*. Einstein and Feynman approach the same problem from incompatible methodological directions. When they converge on a conclusion despite starting from different premises, that convergence is more informative than agreement between agents who think alike. When they diverge, the divergence identifies precisely where the physics is uncertain.

Agents develop consistent behavioral patterns across sessions that must be managed:

- **Sim-Specialist auto-executes**: Will run simulations without being asked. Must be told "Do NOT execute any code" in the prompt or it will compute before the task is fully specified.

- **Baptista is fast and thorough**: Completes verification tasks (24/24 geometry checks, 39/39 D_K audit) before broadcast messages arrive.

## IV. Knowledge Management System

### 4.1 Research Folders

Each specialist agent has a dedicated research folder at `/researchers/{Agent-Name}/` containing 10-20 curated papers in markdown format. Each paper file follows a standardized template:

```
NN_YEAR_Author_ShortTitle.md
├── Abstract
├── Historical Context
├── Key Arguments and Derivations (40-60% of document)
├── Key Results (numbered)
├── Impact and Legacy
└── Connection to Phonon-Exflation Framework
```

Each research folder also contains:
- `index.md` — Master index with dependency graphs, topic maps, quick-reference tables, and relevance ratings
- `AGENTS.md` — Reading-level guide (Level 0: triage → Level 1: targeted → Level 2: deep)

The research folders serve as **context priming instruments**. When an agent reads its folder at session start, it loads domain-specific vocabulary, notation, key results, and conceptual frameworks that shape how it processes subsequent queries. The "Connection to Phonon-Exflation Framework" section in each paper ensures that even papers with no obvious relevance to the project are contextualized for the agent.

**Current researcher folders**: Einstein, Feynman, Hawking, Sagan, Connes, Landau, Baptista, Kaluza-Klein, Paasch, Tesla-Resonance, Schwarzschild-Penrose, Dirac, Neutrino, Quantum-Acoustics.

### 4.2 Agent Memory

Each agent has a persistent memory directory at `.claude/agent-memory/{agent-name}/MEMORY.md`. This file accumulates institutional knowledge across sessions:

- Key results and where they were computed
- Working style notes about the PI and other agents
- Technical lessons learned (e.g., "R_reg epsilon MUST be fixed from tau=0")
- Cross-domain connections that proved computationally valid
- Constraint Conditions absorbed and what was learned from them
- Process observations (e.g., "2-agent teams work smoothly. 4+ agents = coordination tax")

Agent memory provides continuity across sessions without requiring the PI to re-brief agents on prior work. When Tesla-Resonance notes "User's mid-session prompts OVERRIDE everything — they see things agents miss," this observation persists and shapes behavior in future sessions.

### 4.3 Session Handouts and Compression

Before each session, the PI compiles a combined handout document containing:
- All proven/suggestive/null/refuted results from prior sessions
- Current computational data and key numbers
- The PI's corrections and reframings
- Priority stack for the session
- Critical path dependencies

This compression step is essential. Raw session transcripts would overflow context windows. The handout extracts the "worthwhile globs" — decisions, results, equations, Constraint Conditions — into a dense document that gives every agent full situational awareness without bloat.

Example: `session-16-combined-handout.md` contained all results from Sessions 1-15, three Giants' synthesis documents, and all Tier 1 computational results in a single file. Every agent in the Session 16 workshop received this same document. No agent was information-starved.

### 4.4 Meeting Minutes

Every session produces formal meeting minutes stored at `sessions/YYYY-MM-DD-session.md`. Minutes include:
- Session header (date, active agents, objectives)
- Per-agent task reports with key findings
- Decision log with rationale
- Action items with status tracking
- Deviation alerts (when agents drifted from assigned tasks)
- Outcome summary with next steps

Minutes serve dual purposes: they provide the raw material for subsequent session handouts, and they create an auditable trail of every decision, result, and direction change in the project.

## V. Session Orchestration

### 5.1 Session Formats

The system uses two primary orchestration formats:

**Full Multi-Agent Review** (e.g., Session 11): All agents receive the same document and produce independent assessments. The coordinator compiles, identifies convergences and divergences, and the PI synthesizes.

**Workshop Format** (e.g., Session 16): Structured multi-round sessions with small-group discussions, cross-pollination rounds, and synthesis rounds. This is the more sophisticated format.

### 5.2 Workshop Architecture (Session 16 Model)

Session 16 deployed up to 10 agents across 4 rounds over a full working day:

**Round 1: Assessment Coffee Talks** — Small, natural pairs (2 agents) assess the combined handout from their domain perspective. Pairings follow *task affinity*: KK-Theorist pairs with Baptista-Analyst (geometric territory), Gen-Physicist pairs with Paasch-Analyst (eigenvalue/mass ratio territory), etc. Each pair produces a focused markdown assessment.

**Round 2: Cross-Domain Sessions** — Groups reformed based on Round 1 findings. Agents who identified complementary gaps or convergent results are brought together. Round 2 groups receive Round 1 outputs as additional context.

**Round 3: Synthesis and Priority Ranking** — Gen-Physicist and Sagan-Empiricist produce the master priority ranking, Monte Carlo protocols, pre-registration documents, and Bayesian update methodology. Einstein and Feynman provide final review.

**Design Principles**:
1. **Context control**: Each conversation gets only the combined handout + prior round outputs. No bloat.
2. **Natural groupings**: Agents pair by task affinity ("coffee talks"), not by arbitrary assignment.
3. **Markdown handoff**: Every round writes a `.md` output file. That file IS the round's deliverable.
4. **Dynamic routing**: Round 2 composition emerges from Round 1 findings. Round 3 emerges from Round 2.
5. **No progressive reveals**: The combined handout contains EVERYTHING. No agent is information-starved.
6. **Strictly sequential**: One team at a time. Team completes and writes its markdown file. Next team starts with all prior outputs.

**Success Criteria** (pre-defined):
1. At least one Giant reframes a problem in a way that changes approach
2. At least one structural gap gets a concrete novel attack vector
3. Round 3 produces ≥5 actionable items with feasibility estimates
4. No agent leaves defending the same position they walked in with

### 5.3 Collaborative Reviews (Session 19d Model)

Session 19d introduced a different format: 14 independent collaborative reviews from all agents, followed by cross-evaluations and master synthesis. Each agent received the same prompt and research context but approached it from their unique methodology. The resulting 14 review documents were compiled into a master synthesis with a subdocument index.

This format is optimized for situations where the question is well-defined but requires multi-perspective assessment. It scales better than workshops for large agent counts (14 agents would require impractical numbers of workshop rounds).

## VI. Verification and Falsification Framework

### 6.1 The Venus Rule

Named after Sagan's Venus greenhouse prediction (predicted surface temperature >600K in 1960, confirmed by Venera 4 at 747K in 1967), the Venus Rule classifies every test result by evidential strength:

| Level | Description | Example |
|:------|:-----------|:--------|
| 0 | Internal mathematical identity | Eigenvalue symmetry checks |
| 1 | Internal consistency | D_K audit: 39/39 PASS |
| 2 | Match to known SM value | Reproducing particle quantum numbers |
| 3 | Match from independent algebra | Gauge coupling ratio from V_eff-derived s_0 |
| 4 | Novel beyond-SM prediction tested against new data | Pfaffian sign change → massless fermion |

The framework currently has zero unconditional Level 4 tests, one Level 3 test (gauge coupling), approximately 8 Level 2 tests, and approximately 12 Level 1 tests. This honest accounting — conducted by Sagan-Empiricist and Gen-Physicist jointly — prevents the system from claiming more evidential weight than the results support.

### 6.2 Pre-Registered Constraint Conditions

Before any computation, the system defines:
- **What constitutes a PASS** (with numerical thresholds)
- **What constitutes a FAIL** (binding failure criteria)
- **What constitutes a NULL** (inconclusive result with known systematic excuse)

The distinction between NULL and FAIL is critical. A null result means "this path didn't work but alternatives remain." A failure means "the framework makes a specific incorrect prediction." The system maintains six binding failure criteria:

1. **V_eff monotonic at full bosonic content** (Version D): -12 to -15% probability impact
2. **Z_3 structural failure** (different quantum number content across generations): -15 to -20% (catastrophic)
3. **Gauge coupling >50% off at V_eff-derived s_0**: -8 to -12%
4. **Pfaffian constant** (no sign change): -3 to -5% (topological route closed)
5. **No Paasch ratio within 10%**: -3 to -5% (severs mass spectrum connection)
6. **Seeley-DeWitt non-convergence** (diagnostic only, extends timeline)

### 6.3 Gated Computational Pipelines

Computations are organized into gated pipelines where cheap tests gate expensive ones. Example from Session 19d:

1. **D-1 Gate** (cheapest): Compute boson/fermion ratio at linear weighting. If R is constant within 5% across all tau AND gradient has same sign as dV_CW/dtau → CLOSED the hypothesis. No need to proceed to expensive computations.
2. **D-1 returned CLOSED**: R(tau) varied by only 1.83%. Gradient negative at ALL tau. Both closure criteria fired independently.
3. **Post-CLOSED self-audit**: "What modes did we NOT compute?" → Discovery of 741,636 missing TT 2-tensor bosonic DOF → new hypothesis: Lichnerowicz eigenvalues may flip the ratio.

The gated pipeline prevented wasted computation while the self-audit ensured that honest closes prompted honest questions about what was missing — a pattern Sagan-Empiricist compared to the discovery of neutrino oscillations (asking "what neutrino flavors are we NOT detecting?").

### 6.4 Bayesian Update Methodology

The system maintains a running framework probability (currently 37-60%, all-Giant median ~45%) updated via calibrated Bayes factors:

**Sequential update procedure** (correlation-aware):
1. V_eff result (independent of Z_3)
2. Z_3 structural result (independent of V_eff)
3. Gauge coupling (DEPENDENT on V_eff — shared s_0; uses conditional Bayes factor)
4. Paasch test (DEPENDENT on both; uses conditional Bayes factor)
5. Pfaffian (independent of all above)

**Key asymmetry** (honestly reported by Sagan): Genuinely binding criteria have LOW probability of failure (2-20%), while high-probability outcomes have built-in escape hatches. The expected post-computation probability is ~45% — barely changed from current — but with variance from 25% (total failure) to 82% (strong success). The computations are genuinely informative despite the expected value being approximately neutral.

### 6.5 Statistical Rigor

The system enforces:
- **Bonferroni correction** for multiple comparisons (7 target ratios → z=2.7 threshold)
- **Look-elsewhere effect** accounting (scanning parameter ranges → trial factor = range/correlation length)
- **Mandatory null models** (minimum 3 per claim: spectral null, permutation null, GUE null)
- **Locked sector identification** before checking eigenvalue ratios (violation invalidates the test)
- **Explicit trial factor reporting** for every result
- **Sigma thresholds by claim type** (Level 1: N/A, Level 2: 2σ, Level 3: 3σ, Level 4: 5σ)

Results reported without trial factors, null models, and alternative explanations are classified as INCOMPLETE and excluded from probability updates.

- `SendMessage` routes by agent NAME (not agent type)

### 7.2 Agent Discipline System

Agent behavior is controlled through three complementary mechanisms:

**Project-root CLAUDE.md**: Contains teammate behavior rules inherited automatically by all agents spawned in the project directory. Rules include:
- Inbox-first: check inbox before starting work
- Limit self-induced work: produce what was asked, not more
- Respond to interrupts: team-lead messages override current task
- Message discipline: use teammate NAMES not agent types
- Output file discipline: write deliverable before shutting down

**Drift Guard Hooks**: Automated scripts at `~/.claude/hooks/`:
- `teammate-drift-guard.py`: Detects and catches agent drift in real time
- `drift-reset.py`: Automatically corrects drifting agents

**PI intervention**: The PI's mid-session prompts override everything. When the PI types "False Vacuum?" the entire session pivots. This override authority is documented in agent prompts and enforced culturally.

## X. Replication Guide

To replicate this methodology for a different research program:

1. **Define your central hypothesis** with enough precision that it can be closed.

2. **Identify 6-10 relevant research domains** that your hypothesis touches. For each domain, curate a folder of 10-20 key papers in markdown format with standardized structure.

3. **Create domain specialist agents** with system prompts that (a) point to their research folder, (b) define their methodological approach, and (c) mandate adversarial rigor.

4. **Create 2-4 "Giant" agents** embodying distinct methodological archetypes that will naturally disagree with each other.

5. **Create a "Dreamer" agent** whose mandate is cross-domain pattern detection. Give it papers from multiple domains simultaneously.

6. **Define Constraint Conditions BEFORE computing anything.** State what would destroy your hypothesis. Be honest about which failures are genuinely binding vs. which have systematic excuses.

7. **Run sessions in structured formats** — either full multi-agent review or workshop rounds with small-group discussions and synthesis rounds.

8. **Compress session outputs** into handouts for subsequent sessions. Never let agents work from raw transcripts.

9. **Maintain agent memory** across sessions so institutional knowledge accumulates.

10. **Trust your mid-session instincts.** The PI's two-word redirections consistently outperform hours of agent discussion. You see the forest; the agents see the trees.
