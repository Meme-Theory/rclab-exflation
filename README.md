## Session Orchestration

### Format 1: Full Multi-Agent Review

All agents receive the same document, produce independent assessments. Coordinator compiles convergences and divergences. Principal investigator synthesizes.

### Format 2: Workshop (Coffee-Break Rounds)

1. **Round 1 — Assessment Coffee Talks:** Small natural pairs (2 agents) assess combined handout. Pairings by task affinity (e.g., KK + Baptista for geometry).
2. **Round 2 — Cross-Domain Sessions:** Groups reformed based on Round 1 findings. Agents with complementary results brought together.
3. **Round 3 — Synthesis & Priority Ranking:** Gen-Physicist + Sagan produce master priority ranking. Einstein + Feynman provide final review.

Design principles: context control (each conversation gets only handout + prior round outputs), markdown handoff (every round writes a deliverable), strictly sequential (one team at a time).

### Format 3: Collaborative Reviews

14 independent reviews from all agents on the same prompt. Unique methodology per agent. Compiled into master synthesis with subdocument index. Scales better than workshops for large agent counts.

### Compression Strategy

Session outputs are compressed into dense handouts — decisions, results, equations, constraint conditions — so every agent receives full situational awareness without bloat. Example: Session 16's handout contained all results from Sessions 1-15, three Giants' synthesis documents, and all Tier 1 computational results in a single file.

## Replication Guide

To replicate this methodology for a different research program:

1. **Define central hypothesis** with enough precision to be closable
2. **Identify 6-10 relevant research domains** — curate 10-20 key papers per domain in markdown
3. **Create domain specialist agents** with system prompts pointing to research folders, defining methodology, mandating rigor
4. **Create 2-4 "Giant" agents** embodying distinct methodological archetypes
5. **Create a "Dreamer" agent** for cross-domain pattern detection
6. **Define constraint conditions BEFORE computing** — what destroys your hypothesis?
7. **Run sessions in structured formats** (full review, workshop rounds, or collaborative reviews)
8. **Compress session outputs** into handouts — never let agents work from raw transcripts
9. **Maintain agent memory** across sessions for institutional knowledge
10. **Trust mid-session operator instincts** — two-word redirections consistently outperform hours of agent discussion

## Epistemological Contribution

The deepest insight of this project is not about physics. It is about what LLMs actually are when used this way.

**Standard framing:** LLMs are "just pattern matching."

**Actual truth:** Mathematics IS pattern matching. The unreasonable effectiveness of mathematics in physics is precisely the claim that patterns recur across domains.

The training data already contains all of Connes and all of Volovik and all of Baptista and all of random matrix theory and all of structural acoustics. These sit in separate compartments because no one asks the question that connects them.

This system creates contexts where cross-domain connections are forced to surface. Tesla sees acoustic impedance in the Lichnerowicz operator because acoustic physics papers and KK geometry papers are placed in the same context with the mandate: "What resonates?"

If the connections are real, the system finds real physics. If not, constraint conditions fire.

The constraint conditions keep not firing... but the stabilization mechanisms keep closing. 2-4% is not proof. But it is not nothing, the methodology is sound, the next computation is defined, and the mathematics stands on its own regardless of whether the cosmological interpretation survives.
